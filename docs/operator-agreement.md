# Operator Agreement — TEMPLATE STUB

> **TEMPLATE STUB — binding operator policy; finalize with counsel before any operator signs.**
> This is a structural template, **not** an executable agreement. Every commitment below is a
> `[TEMPLATE — counsel to finalize]` stub. The operator agreement is a **binding** part of the

The white-label operator signs the following seven commitments (headers derive from
`.planning/research/PITFALLS.md` §"Operator-facing checklist" — referenced, not re-authored as
finished legal prose).

## 1. Will not enable token purchase

- `[TEMPLATE — counsel to finalize]` The operator will not enable any mechanism for users to
  purchase tokens.

## 2. Will not enable token redemption

- `[TEMPLATE — counsel to finalize]` The operator will not enable any mechanism for users to
  redeem tokens for value.

## 3. Will not award cash or prizes based on bet outcomes

- `[TEMPLATE — counsel to finalize]` The operator will not award cash or prizes of monetary value
  based on bet outcomes.

## 4. Will geo-fence per the recommended list

- `[TEMPLATE — counsel to finalize]` The operator will enforce geo-fencing per the recommended
  jurisdiction list. The list itself is `[DEFERRED: external counsel]` (see `docs/regulatory.md`).

## 5. Will display a ToS asserting tokens have no monetary value

- `[TEMPLATE — counsel to finalize]` The operator will display a Terms of Service asserting that
  tokens have no monetary value (see `docs/terms-of-service.md`).

## 6. Will not advertise tokens as having monetary value

- `[TEMPLATE — counsel to finalize]` The operator will not advertise or represent tokens as having
  monetary value.

## 7. Will obtain a DGOJ license before any real-money plan

- `[TEMPLATE — counsel to finalize]` The operator will obtain a DGOJ license before any "convert to
  real money" plan.
