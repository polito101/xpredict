<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client form → use-server action | The admin_jwt HttpOnly cookie is read SERVER-SIDE only; it never reaches client JS. |
| use-server action → FastAPI admin endpoint | Authenticated multipart PUT crosses with operator-controlled palette + logo. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-11 | Information Disclosure | admin Bearer token leaking to client JS | mitigate | The Bearer is forwarded only inside the `"use server"` branding-admin-api.ts (cookies() server-side); the client component calls the exported async action and cannot read the HttpOnly cookie — same proven pattern as admin-api.ts. |
| T-10-12 | Tampering | client-only hex/logo validation bypassed | mitigate | The client zod/size checks are UX-only; the server (Plan 10-01) is authoritative (D-09). A 422 from the server maps to inline FormMessage — the client never assumes its own validation is the gate. |
| T-10-13 | Elevation (XSS) | logo object-URL preview executing script | mitigate | The preview uses `URL.createObjectURL` rendered via `<img src>` (SVG-in-img does not execute script); the file is never inlined into the DOM as markup. |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No new packages this phase (RESEARCH §Standard Stack — all frontend deps already present; zero new installs). No legitimacy checkpoint required. |
</threat_model>
