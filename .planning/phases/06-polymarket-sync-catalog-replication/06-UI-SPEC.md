## Layout Contract

### Home page (`frontend/src/app/page.tsx` -- replace scaffold placeholder)

```
+----------------------------------------------------------+
|  [Nav bar - Phase 9+ concern, not in this phase]         |
+----------------------------------------------------------+
|                                                          |
|  Markets                              [sort indicator]   |
|                                                          |
|  +----------------+  +----------------+  +------------+  |
|  | Market Card    |  | Market Card    |  | Market Card|  |
|  | [question]     |  | [question]     |  | [question] |  |
|  | YES 63% NO 37% |  | YES 48% NO 52% |  | YES 80%...|  |
|  | Vol: $2.1M     |  | Vol: $1.8M     |  | Vol: $900K |  |
|  | Dec 31, 2026   |  | Jan 15, 2027   |  | Nov 1, 20..|  |
|  |     [House]    |  |  [Polymarket>] |  | [Polymrkt>]|  |
|  +----------------+  +----------------+  +------------+  |
|                                                          |
|  +----------------+  +----------------+  +------------+  |
|  | ...            |  | ...            |  | ...        |  |
|  +----------------+  +----------------+  +------------+  |
|                                                          |
+----------------------------------------------------------+
```

- **Grid:** CSS Grid, 3 columns on desktop (>=1024px), 2 columns on tablet (>=640px), 1 column on mobile (<640px)
- **Max width:** `max-w-6xl` centered (matches admin layout `max-w-6xl`)
- **Page padding:** `px-6 py-12` (24px horizontal, 48px top)
- **Grid gap:** `gap-4` (16px)
- **Card minimum height:** Not fixed -- content-driven; question text truncated at 3 lines via `line-clamp-3`

### MarketCard anatomy

```
+----------------------------------------+
| CardHeader (p-6 pb-2)                  |
|   [Question text - line-clamp-3]       |
|                                        |
| CardContent (p-6 pt-0)                 |
|   +----------------------------------+ |
|   | YES 63%              NO 37%      | |
|   +==================================+ |
|   | [===emerald=======][===rose===]  | | <- odds bar
|   +----------------------------------+ |
|                                        |
| CardFooter (p-6 pt-0)                 |
|   Vol: $2.1M   |   Dec 31, 2026       |
|                          [Polymarket>] |
+----------------------------------------+
```

- **Odds bar:** Horizontal bar, full width, split proportionally between YES (emerald-500) and NO (rose-500), height 6px, rounded corners (`rounded-full`)
- **Question text:** `text-base font-semibold leading-snug line-clamp-3` -- 3-line max with ellipsis
- **Metadata row:** Flex with `justify-between`, volume left, deadline right, `text-sm text-zinc-500`
- **Source badge:** Positioned in CardFooter, right-aligned, below metadata row
- **Card hover:** `hover:shadow-md transition-shadow` -- subtle elevation on hover
- **Card click area:** Entire card is a link to `/markets/{slug}` (Phase 9 detail page; link renders but page will 404 until Phase 9)

### Odds display

- YES and NO percentages displayed as integers: `Math.round(odds * 100)` + `%`
- YES text uses `text-emerald-700 dark:text-emerald-400 font-medium`
- NO text uses `text-rose-700 dark:text-rose-400 font-medium`
- Layout: flex row with `justify-between`, labels "YES" and "NO" as `text-xs uppercase tracking-wide text-zinc-500`

### Volume formatting

- `>= 1_000_000`: `$X.XM` (e.g., `$2.1M`)
- `>= 1_000`: `$X.XK` (e.g., `$450K`)
- `< 1_000`: `$XXX` (e.g., `$89`)
- Always prefix with `$` (play money uses USD-equivalent display)

### Deadline formatting

- Use `Intl.DateTimeFormat` with `{ month: 'short', day: 'numeric', year: 'numeric' }`
- Example: `Dec 31, 2026`
- Past deadline: show `Ended` in `text-zinc-400` instead of date

## States Matrix

| State | Trigger | Visual |
|-------|---------|--------|
| Loading | Initial page load / data fetch in progress | `MarketListSkeleton`: grid of 6 skeleton cards matching card dimensions (3x2 on desktop) |
| Empty | API returns 0 markets (no house + no mirrored) | Centered illustration-free block: heading + body from copywriting table, `text-zinc-500`, vertically centered in content area |
| Populated | API returns 1+ markets | `MarketList` grid with `MarketCard` components |
| Error | API fetch fails (network error, 5xx) | Centered error block: heading + body from copywriting table, `text-rose-700` heading, "Try refreshing" as actionable guidance |
| Card hover | Mouse enters card | `shadow-md` transition (200ms ease) |
| Stale data | Server-side render; no real-time in this phase | No staleness indicator -- WebSocket arrives in Phase 9 |

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Card click | Navigate to `/markets/{slug}` (full card is a link via `<Link>` wrapping) |
| Source badge "Polymarket" click | Opens Polymarket source URL in new tab (`target="_blank" rel="noopener noreferrer"`); click does NOT propagate to card link |
| Card hover | Elevate shadow from `shadow-sm` to `shadow-md` over 200ms |
| Page refresh | Full server-side re-render; shows skeleton during fetch |
| Keyboard focus | Card receives visible `ring-2 ring-zinc-950 ring-offset-2` focus ring (matches button focus from existing components) |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Card link semantics | Each `MarketCard` wraps content in `<Link>` or `<a>` with `aria-label="{question}"` |
| Source badge external link | Polymarket badge includes `aria-label="View on Polymarket (opens in new tab)"` |
| Odds bar | `role="img"` with `aria-label="YES {X}%, NO {Y}%"` on the visual bar |
| Color contrast | All text/background combinations meet WCAG 2.1 AA (4.5:1 for body, 3:1 for large text) |
| Reduced motion | `prefers-reduced-motion: reduce` disables hover shadow transition |
| Empty/error states | Use `role="status"` (matches existing auth page pattern from Phase 2) |
| Skeleton loading | `aria-hidden="true"` on skeleton elements; `aria-busy="true"` on parent container |

## Dark Mode

Follow existing project pattern: CSS variables in `globals.css` with `@media (prefers-color-scheme: dark)`. All color tokens above include dark variants. No manual toggle in this phase.

Key dark mode mappings:
- Page background: `bg-zinc-950` (existing)

- Metadata text: `text-zinc-400`
- YES odds: `text-emerald-400`
- NO odds: `text-rose-400`
- House badge: inverted to `bg-zinc-100 text-zinc-900`
