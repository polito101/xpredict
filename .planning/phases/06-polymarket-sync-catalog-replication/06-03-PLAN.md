<objective>
Build the first player-facing market list UI on the home page, replacing the Phase 1 scaffold
placeholder. Server Component fetches from /api/v1/markets and renders a responsive grid of
MarketCard components with question, odds bar, volume, deadline, and source badge.

This plan delivers what the player actually sees: the home page with real market data. It can
be developed in parallel with Plan 02 (Celery tasks) because the frontend consumes the public
API endpoint that already exists (Plan 01 provides the adapter and schema; Plan 02 modifies the
endpoint query -- but the response shape is compatible for development).

Covers MKT-01 (market list on home page), MKT-02 (card with question, odds, deadline, volume,
source badge per D-02).

Per D-01: house markets appear first (the API handles sorting; frontend renders in order).
Per D-02: badge is a small chip bottom-right of card; Polymarket badge links to polymarket.com;
House badge has no link.

Purpose: First player-visible output of Phase 6 -- the market catalog becomes browseable.

Output: 5 new components, modified home page, API fetch helper, component tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backend API -> Frontend SSR | Market data including question text and source_url rendered in HTML |
| Frontend -> Browser | External links (Polymarket URLs) rendered as anchor tags |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-09 | Tampering | Market question text (XSS) | mitigate | React auto-escapes JSX text content; question is rendered as text node in h3, never dangerouslySetInnerHTML; Pydantic validates server-side |
| T-06-10 | Information Disclosure | source_url external link | mitigate | URL is derived server-side from source_market_id (not user input); rendered with rel="noopener noreferrer" and target="_blank"; no server-side fetch of the URL |
| T-06-11 | Spoofing | fetchMarkets API call | accept | Server Component fetch runs on the server (not browser); connects to internal API; no user auth required for public market list (read-only data) |
| T-06-SC | Tampering | npm installs | mitigate | Only shadcn official registry components (badge, skeleton) added; lucide-react is shadcn default icon library; all from official npm registry |
</threat_model>
