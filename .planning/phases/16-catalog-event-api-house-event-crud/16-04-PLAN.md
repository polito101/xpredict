<objective>
Expose the Phase-15 `EventService.resolve_event/void_event/reverse_event` over HTTP (the EVA-03..06 HTTP surface) by extending `event_admin_router` with `POST /admin/events/{group_id}/resolve|void|reverse`. Each endpoint implements the stateless two-step confirm (`confirm:false`/absent → 200 non-mutating preview; `confirm:true` → execute via the service), validates a mandatory non-empty justification (and a child-YES `winning_outcome_id` for resolve), maps the service's `ValueError`s to HTTP (mirrored 409, blank justification 422, bad winning-outcome 422, missing 404), and follows the settlement session choreography (capture `admin_id` then `session.rollback()` before the service owns its per-child sessions).

Purpose: Phase 15 shipped the settlement engine; this plan is the thin HTTP glue that exposes it with the agreed stateless preview/confirm and the exact error contract — reusing the service byte-for-byte, never re-implementing the per-child settle loop (the 23505 dangling-tx landmine).
Output: `app/settlement/event_schemas.py` (extended with the request/response models), `app/settlement/event_router.py` (extended with the three routes + the ValueError map), `tests/settlement/test_event_settle_router.py`. Router registration in `main.py` is plan 16-05.
</objective>
