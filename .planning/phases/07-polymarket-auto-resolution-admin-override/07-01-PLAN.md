# Plan 07-01: Schema Foundation + Wave 0 Tests

## Objective
Add the `markets.uma_resolved_at` grace-period column (ORM field + Alembic migration 0007), the `POLYMARKET_GRACE_PERIOD_MINUTES` setting, and the Wave 0 test scaffolds that Plans 07-02 and 07-03 will fill in.

## Context
This is the foundation wave for Phase 7. Per `07-RESEARCH.md` (Pattern 1) the grace-period state for Polymarket auto-resolution MUST live in a DB column (not Redis) so it survives Celery at-least-once delivery and worker restarts (constraint 2). Plan 07-02 (auto-resolution Beat task) and Plan 07-03 (admin force-settle endpoint) both depend on this column existing and on the migration being applied. The Wave 0 test files are created here as scaffolds (collectable, skipped/xfail) so the downstream plans have a fixed target file path to fill in — this satisfies the Nyquist rule that every behavior have a test file before implementation. No new external packages are introduced (entire phase is additive).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Alembic migration → Postgres schema | DDL change crosses into the persistent store; must be reversible and non-destructive |

## Security Threats

| Threat ID | Threat | STRIDE | Disposition | Mitigation |
|-----------|--------|--------|-------------|------------|
| T-07-01 | Migration drops or rewrites existing market data | Tampering | mitigate | `op.add_column` only — nullable, no `server_default`, no backfill, no `NOT NULL`; existing rows get NULL automatically. `downgrade()` is a pure `drop_column`. |
| T-07-02 | Grace-period column made non-nullable, breaking existing rows on upgrade | Denial of Service | mitigate | Column is explicitly `nullable=True`; NULL is the documented safe default meaning "no grace period in progress" (constraint 15). |
| T-07-SC | Supply-chain: new package pulled in during this plan | Tampering | accept | No new packages introduced in Phase 7 (RESEARCH Package Legitimacy Audit: none); no install task in this plan, so no slopcheck checkpoint required. |
</threat_model>
