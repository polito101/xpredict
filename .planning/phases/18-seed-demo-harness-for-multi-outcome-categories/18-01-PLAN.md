# Plan 18-01 — Seed/Demo Harness for Multi-outcome + Categories

## Goal

One command (`uv run python bin/seed_demo.py`) seeds a credible multi-outcome demo across all 7
canonical categories and all 4 event states, with non-flat per-outcome odds history; `--reset` is
idempotent; the spike-004 double-entry integrity check is green after seed and after reset.

**Scope:** `backend/bin/seed_demo.py` (extend), `backend/tests/seed/test_seed_demo_e2e.py` (extend),
`Makefile` (targets). **No new domain code, no migration.** Everything rides the merged
EventService / SettlementService / Wallet / Bet service layer.

## Tasks

### Task 1 — Reset-set fix + canonical category model + event specs
**Commit:** `feat(18): reset market_groups + canonical retag + multi-outcome event specs`

1. Add `"market_groups"` to `_RESET_TABLES` (DEMO-04 idempotency — D3).
2. Retag `_MARKET_TEMPLATES` categories to canonical (Space→World, Climate→World,
   Entertainment→Pop Culture, Commodities→Economy, Finance→Economy) — D5.
3. Add `FEATURED_CATEGORIES: tuple[str,...]` (the 7 canonical names, pinned) and
   `MIN_ITEMS_PER_FEATURED_CATEGORY = 2`; add `_assert_featured_categories_match_canonical()`
   comparing against `get_settings().POLYMARKET_CATEGORIES` (D4).
4. Add `EventOutcomeSpec`, `DemoEventSpec` (frozen dataclasses) + `_EVENT_TEMPLATES` (7 events,
   ordered: resolved, partial, void, then 4 open; 3–8 outcomes each; independent plausible YES odds;
   deterministic slugs `demo-evt-{NN}-{cat-slug}`).
5. `SeedConfig` gains `n_events: int = 7`; add pure `build_event_specs(cfg) -> list[DemoEventSpec]`
   (first `n_events` templates) + `SeededEvent` dataclass.

### Task 2 — Seed events: create, odds, bets, 4-state resolution + orchestration
**Commit:** `feat(18): seed multi-outcome events end-to-end + integrity check + make targets`

1. `seed_events(cfg) -> list[SeededEvent]` — per spec, `EventService.create_house_event` on a fresh
   session; read back each child's id/slug/YES+NO outcome ids (mirror `seed_markets`); represent
   children as `SeededMarket` (resolve_to=None) so odds reuse works.
2. Odds: build the combined chartable list (standalone `markets` + all event children) and pass to the
   existing `seed_odds_history` (non-flat per-outcome series — DEMO-02).
3. `seed_event_bets(cfg, users, events) -> int` — deterministic both-sides spread per child via
   `BetService.place_bet` (session-per-bet) so resolved/void/partial show winners AND losers.
4. `seed_event_resolutions(cfg, events) -> int` — per `target_state`: `open`→none;
   `resolved`→`EventService.resolve_event(winner child YES id)`; `void`→`EventService.void_event`;
   `partially_resolved`→`SettlementService.resolve_market` on ONE child (D2).
5. Wire into `seed_demo` orchestration (users→markets→events→odds(all)→bets+event_bets→
   resolutions+event_resolutions); extend `SeedResult` (events, event_bets, event_resolutions,
   featured_categories). Call `_assert_featured_categories_match_canonical()` at seed start.
6. `verify_integrity() -> dict[str,int]` wrapping `_reconcile_async`; CLI `main()` prints
   `integrity: accounts=N drift=M` after both seed and reset paths (DEMO-04 surfaced).
7. `Makefile`: repoint `seed` → real seed; add `demo-reset` → `--reset`; update help + `.PHONY` (D9).

### Task 3 — Tests: 4 states, featured categories, reset idempotency
**Commit:** `test(18): event seed coverage — states, categories, reset idempotency, integrity`

1. Pin the 3 existing tests to `n_events=0` (keep standalone-focused/fast).
2. `test_seed_events_cover_all_four_states` — `n_markets=0, n_events=7`; query each group's children →
   `derive_event_status`; assert `{open, partially_resolved, resolved, void}` all present; reconcile baseline-relative.
3. `test_seed_events_fill_featured_categories` — `n_markets=15, n_events=7`; `CatalogService.list_categories`
   ⊇ `FEATURED_CATEGORIES`; each featured category `list_catalog(category=…)` len ≥ `MIN_ITEMS_PER_FEATURED_CATEGORY`; reconcile green.
4. `test_reset_clears_market_groups_and_reseeds` — seed events, `reset_demo()`, re-seed events → no slug
   collision (re-seed succeeds), event groups present, reconcile baseline-relative (DEMO-04).
5. `test_seed_events_non_flat_odds_history` — assert ≥1 event child has a non-flat OddsSnapshot series (DEMO-02).

## Risks / Mitigations

- **23505 dangling-tx** → session-per-call for every self-committing service (mirrors the existing spine).
- **resolve_event CR-01** → pass the read-back YES outcome id of the chosen winner child.
- **Test runtime** (event bets are the slow part) → minimal both-sides spread per child; state test uses `n_markets=0`.
- **Windows worktree backend flake** (testcontainers/ruff) → validate per-module locally; trust Linux CI for the full suite (memory: [[xprediction-backend-fullsuite-testcontainers-flake]]).
