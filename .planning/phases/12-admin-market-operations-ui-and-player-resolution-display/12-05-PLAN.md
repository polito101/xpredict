| admin browser -> /api/v1/admin/markets | all market CRUD is admin-Bearer-gated server-side; the client form/table only forward the admin_jwt via the use-server layer |
| admin form input -> create/patch | the zod mirror is UX-only; the FastAPI MarketCreate/MarketUpdate Field constraints are authoritative |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-14 | Elevation of Privilege | non-admin reaching market CRUD | accept (backend-owned) | `current_active_admin` gates every CRUD route (401/403, tested in `test_admin_router.py::test_create_market_non_admin_returns_403`); this plan adds no new endpoint, only the cloned client layer |
| T-12-15 | Tampering | client bypassing field constraints (e.g. criteria after first bet) | mitigate | The backend enforces `MarketCreate`/`MarketUpdate` Field bounds + returns 423 CRITERIA_LOCKED after the first bet; the form's disable + 422 mapping is a UX mirror, never the authority |
| T-12-16 | Tampering | money (stake limits) as float | mitigate | Min/Max stake stay strings; `Number(...)` only for the client min<=max compare; the backend stores `Numeric(18,4)` (12-01) and money-lint enforces it |
| T-12-17 | Information Disclosure | admin_jwt to client JS | mitigate | All CRUD calls funnel through the `"use server"` `admin-markets-api.ts` (12-02) reading the HttpOnly cookie server-side |
| T-12-SC | Tampering | npm/pip/cargo installs | mitigate | Zero installs (RESEARCH §Package Legitimacy Audit); no install task present |
