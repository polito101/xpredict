Build the shared frontend foundation every admin slice depends on: the `"use server"` Bearer-forwarding API layer for market CRUD + settlement actions, the Wave-0 URL-prefix contract guard (the single most important new test), the shared 5-state market status badge, and a status-badge test. This plan clones that guard for the new market wrappers so the regression is structurally impossible, and centralizes the api/types/badge so Wave-2 has zero cross-plan file collisions.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser -> Next server action | admin_jwt is an HttpOnly cookie; the wrapper reads it server-side and forwards a Bearer header — the token must never cross into client JS |
| Next server action -> FastAPI admin endpoints | the two router prefixes differ; a wrong prefix silently 404s, which can mask an auth bug |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-05 | Information Disclosure | admin_jwt leaking to client JS | mitigate | `"use server"` module reads the HttpOnly cookie server-side via `bearerHeader`; only the JSON result crosses to the client (cloned admin-api.ts pattern) |
| T-12-06 | Tampering / Repudiation | wrong-prefix call → silent 404 masking an auth failure | mitigate | `admin-markets-api.test.ts` URL-contract guard asserts CRUD keeps `/api/v1` and settlement is bare with `not.toContain("/api/v1")` |
| T-12-07 | Elevation of Privilege | non-admin reaching market mutations | accept (backend-owned) | Every endpoint is gated by `current_active_admin` (401/403, tested in test_admin_router.py / test_settlement_router.py); this plan only adds the client wrappers, no new endpoint |
| T-12-SC | Tampering | npm/pip/cargo installs | mitigate | Zero installs (RESEARCH §Package Legitimacy Audit); no install task present |
</threat_model>
