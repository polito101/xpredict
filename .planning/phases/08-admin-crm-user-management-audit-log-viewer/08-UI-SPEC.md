### User detail page

| State | Trigger | Visual |
|-------|---------|--------|
| Loading | Initial page load | Skeleton blocks for header fields + skeleton table in active tab |
| Loaded: active user | User data loaded, `banned_at` is null | "Ban User" button in `destructive` variant, recharge form enabled |
| Loaded: banned user | User data loaded, `banned_at` is not null | "Unban User" button in `outline` variant, recharge form disabled with tooltip, `UserStatusBadge` shows "Banned" with ban date |

| Recharge submitting | Admin submits recharge form | Submit button shows spinner, disabled |
| Recharge success | API returns 200 | Toast "Wallet recharged successfully", balance updates, new entry appears in transaction table |
| Recharge error | API returns 4xx/5xx | Toast "Failed to recharge wallet", form stays populated |
| Tab switching | Admin clicks a tab | Tab content swaps; each tab manages its own pagination state independently |

## Responsive Behavior

Phase 8 is admin-only UI. Per PLT-07 in REQUIREMENTS.md: "admin UI desktop-only acceptable." No mobile breakpoint optimization is required. However, tables must remain usable down to 1024px viewport width.

| Breakpoint | Behavior |
|------------|----------|
| < 1024px | Tables use horizontal scroll (`overflow-x-auto`) inside a container. Filter bar wraps naturally via flex-wrap. Layout remains functional but not optimized. |
| >= 1024px | Full table display, all columns visible, filter bar in single row |
| >= 1280px | Comfortable spacing, `max-w-6xl` centered content |

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| User row click | Navigate to `/admin/users/{id}` (entire row is clickable via `cursor-pointer`) |
| Search input | Debounced 300ms; triggers server-side refetch with `?search=` query param; clears pagination to page 1 |
| Filter change | Triggers server-side refetch immediately; clears pagination to page 1 |
| Sort column click | Toggles sort direction (asc -> desc -> none); server-side ORDER BY; retains current page |
| Pagination prev/next | Server-side page change; scroll to top of table |
| Ban button click | Opens `BanConfirmDialog`; button does NOT directly ban |
| Ban confirm (dialog) | Submits ban request; dialog stays open during submission (prevent double-click); closes on success |
| Unban button click | Opens `UnbanConfirmDialog` |
| Recharge submit | Calls `POST /admin/wallets/{user_id}/recharge` with `Idempotency-Key` header (UUID v4 generated client-side per submission) |
| Export CSV click | Opens dropdown; selecting an option triggers `fetch()` to export endpoint with current filter params; browser downloads via `blob` URL |
| Audit payload toggle | Click on chevron or preview text toggles expanded/collapsed; `aria-expanded` attribute tracks state |
| Back to users link | `router.push('/admin/users')` preserving query params if possible (best-effort) |
| Tab switch | Client-side tab change; does NOT change URL (no deep-linking to tabs in v1) |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Table semantics | Use `<table>`, `<thead>`, `<tbody>`, `<th scope="col">`, `<td>` -- shadcn `Table` provides these |
| Sortable columns | `<th>` includes `aria-sort="ascending"` / `"descending"` / `"none"` attribute |
| Row links | Clickable rows use `role="link"` with `aria-label="View user {email}"` |
| Dialog focus trap | Radix `Dialog` handles focus trap automatically; first focusable element receives focus on open |
| Dialog cancel | `Escape` key closes dialog (Radix default behavior) |
| Status badge | `aria-label="Status: Active"` or `"Status: Banned"` on each badge |
| Toast notifications | `sonner` provides `role="status"` with `aria-live="polite"` |
| Pagination | Prev/Next buttons include `aria-label="Previous page"` / `"Next page"`; disabled state uses `aria-disabled="true"` |
| Search input | `aria-label="Search users by email or display name"` (user list) or `"Search audit log by actor"` (audit log) |
| Expanded payload | `aria-expanded="true/false"` on toggle control; content uses `role="region"` with `aria-label="Audit event payload"` |
| Color contrast | All text/background combinations meet WCAG 2.1 AA (4.5:1 for body text, 3:1 for large text and UI components) |
| Reduced motion | `prefers-reduced-motion: reduce` disables row hover transitions and toast slide animations |
| Loading skeletons | `aria-hidden="true"` on skeleton elements; `aria-busy="true"` on parent container |
