<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin Bearer JWT -> export/audit endpoints | All endpoints must verify is_admin=true |
| CSV content -> user's Excel/Sheets | Formula injection risk when opening CSV in spreadsheet |
| Audit log -> admin viewer | Read-only; no mutation path exists at UI or API level |
| Actor search -> SQL ILIKE | Same wildcard injection risk as user search |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Tampering | CSV formula injection | mitigate | sanitize_csv_cell prefixes formula-trigger chars with single quote (D-09); negative test with "=SUM" cell |
| T-08-06 | Elevation of Privilege | Export endpoints auth | mitigate | current_active_admin on all 3 export endpoints; negative auth tests |
| T-08-07 | Elevation of Privilege | Audit log mutation | mitigate | Router has only GET endpoints; DB trigger + REVOKE from Phase 1 blocks UPDATE/DELETE at Postgres level |
| T-08-08 | Information Disclosure | ILIKE wildcard in actor search | mitigate | Escape % and _ in actor search param before ILIKE |
| T-08-09 | Denial of Service | Unbounded CSV export | mitigate | MAX_EXPORT_ROWS=10000 cap with structlog warning |
| T-08-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages in this plan (Python stdlib csv + io only) |
</threat_model>
