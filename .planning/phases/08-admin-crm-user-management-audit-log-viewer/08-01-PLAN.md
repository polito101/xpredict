<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin Bearer JWT -> FastAPI admin endpoints | All admin endpoints must verify is_admin=true via current_active_admin |
| Player cookie -> admin endpoints | Players must NOT access admin endpoints (403) |
| Admin action -> target user data | Admin can view/modify other users' data (authorized by is_admin) |
| Search input -> SQL ILIKE | User-supplied search string enters SQL via ILIKE (wildcard injection risk) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Elevation of Privilege | All /admin/* endpoints | mitigate | current_active_admin dependency on every endpoint + negative auth tests in test_auth_negative.py |
| T-08-02 | Elevation of Privilege | Ban bypass via stale session | mitigate | Check banned_at at 3 enforcement points: login (auth/manager.py), bet (bets/router.py existing), recharge (wallet/admin_router.py) |
| T-08-03 | Information Disclosure | ILIKE wildcard injection | mitigate | Escape % and _ in search input before passing to ILIKE in AdminUserService.list_users |
| T-08-04 | Spoofing | Missing Bearer returns 200 | mitigate | 401 without token, 403 with non-admin token; negative tests assert both |
| T-08-SC | Tampering | npm/pip/cargo installs | mitigate | No new Python packages in this plan; slopcheck gate deferred to Plan 03 (frontend installs) |
</threat_model>
