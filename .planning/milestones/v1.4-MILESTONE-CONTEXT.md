# v1.4 Premium Experience — Milestone Context (plan-of-record)

**Status:** Shipped 2026-06-06 · **Type:** Frontend-only redesign · **Phase:** 19 · **PR:** [#33](https://github.com/polito101/xpredict/pull/33) (`2b2fca8`)

## What

Reposition XPrediction as a **white-label, API-first prediction-market platform** through a premium, frontend-only redesign — a dark-first "Obsidian & Spark" design system, a platform-first public landing, the live app moved behind authentication, and a premium-restyled backoffice. **No backend / API / settlement / catalog / auth / wallet / portfolio / contract changes** (single exception: a pure `ruff format` of the already-merged v1.3 livebets files to unblock CI).

## Locked decisions

1. **Frontend-only** — built entirely against the existing v1.2/v1.3 contracts; zero backend domain churn.
2. **Visible brand = "XPrediction"** everywhere (metadata, navbar, footer, landing, auth, admin). **Technical names stay `XPredict/xpredict`** — cookie `xpredict_session`, `BACKEND_URL`/`NEXT_PUBLIC_*` env vars, repo, internal identifiers unchanged (no contract churn). Brand is runtime-driven (`/branding/current` → `brand_name`), legacy default `"XPredict"`/empty mapped to `"XPrediction"` for display; real operator names still override (white-label intact).
3. **Platform-first landing** at `/`; the app (catalog) moves to `/markets` behind edge-middleware auth.
4. **Dark-first** semantic token system (Tailwind v4 `@theme inline`); identity primitives `XMark`/`Spark`/`Aurora`; Space Grotesk + Inter.
5. **Invariants are non-negotiable:** white-label runtime branding pipeline, money/odds as display strings, per-outcome framing-LOCK (independent YES bars, never sum-to-100), single live-socket cap, escaped justification, A-LOSS-NEUTRAL (loss neutral, never red).

## Scope

- **Design system:** dark semantic tokens + gradient/glow/aurora utilities; all 17 shadcn primitives retoned.
- **Landing (`/`):** Hero (node-graph) → Pillars (run/integrate/launch) → Capabilities → API section → Demo (real `/catalog` stats + featured) → How it works → CTA; resilient to a down backend.
- **App behind auth:** edge middleware gates `/markets,/events,/portfolio,/wallet,/live` → `/login`; `/admin/*` → `/admin/login`; nav splits logged-out vs in.
- **Player premium:** market/event detail, oversized live odds, dark gradient charts, portfolio performance hero, wallet big-number balance.
- **Admin:** dark-first brand-aware shell (no double header) + tokenized backoffice + dark charts.

## Backend integration (handoff to Pol — carried as deferred, NOT a milestone blocker)

The frontend is complete + correct against the v1.2/v1.3 contracts. To light up real data on the definitive backend:
1. Point env: `BACKEND_URL` (server), `NEXT_PUBLIC_API_URL` (browser), `NEXT_PUBLIC_WS_URL` (live odds); backend CORS/WS origin (`FRONTEND_BASE_URL`) must equal the frontend origin.
2. Set `tenant_config.brand_name = "XPrediction"` (one row, or via `/admin/branding`).
3. Drop the official logo PNG at `frontend/public/brand/xprediction-logo.png` (falls back to a faithful vector mark until then — nothing broken).
4. Seed/verify demo data (+ a verified player + an admin).
5. `/live`: provide the v1.3 live-bets backend + widget env, else it stays in the correct empty state.

Full per-screen backend-dependency matrix + the integration runbook: [`v1.4-phases/phase-19-premium-experience/HANDOFF.md`](v1.4-phases/phase-19-premium-experience/HANDOFF.md).

Any backend/API/contract change · portfolio position market-name/link (needs a small future backend read) · real operator onboarding.
