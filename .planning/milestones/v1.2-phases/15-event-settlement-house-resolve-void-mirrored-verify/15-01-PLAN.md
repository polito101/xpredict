<objective>
Ship the EVT-06 derived event-status read-projection: a pure, stdlib-only `derive_event_status(children)` free function plus its `ChildStatus` input dataclass, both in the NEW module `backend/app/settlement/event_service.py`, with exhaustive unit tests in `backend/tests/settlement/test_derive_event_status.py`.

It deliberately carries NO database I/O and NO stored column — event status is computed at read time from the constituent markets' states (EVT-06 forbids an authoritative `status`/`winning_outcome` column on `market_groups`; migration 0011 deliberately omitted them).

Purpose: Establish the column-free status contract first, in isolation, so it is trivially unit-testable without a session and so Wave 2's `EventService` class can extend this same module.
Output: `event_service.py` (pure layer only — the `EventService` class arrives in Wave 2) + a no-Docker unit test file.
