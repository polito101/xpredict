| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Tampering | `GammaEvent`/`GammaTag`/`GammaEventMarket` model_config | mitigate | `_gamma_model_config()` `extra="ignore"`(dev)/`"allow"`(prod) — injected/unknown fields never bind to model attrs (spike-002 T-06-01). Reused verbatim, not re-implemented. |
| T-14-02 | Tampering / RCE | stringified-JSON child fields (outcomes/outcomePrices/clobTokenIds) | mitigate | Inherited `parse_stringified_json_list` uses `json.loads` ONLY (never `eval`); `JSONDecodeError`→`[]`. Subclass inherits the validator — no new parse path. |
| T-14-03 | Tampering | event-level `volume24hr`/`volume` (FLOAT) → floor input | mitigate | `_safe_decimal(str(value))` in the `*_decimal` properties; never the `volumeNum` float on children. Decimal precision preserved for the downstream $10k floor. |
| T-14-04 | Denial of Service | malformed event element | mitigate | Per-element `model_validate` is wrapped by the caller (adapter/task) in try/except (14-03/14-04); a poisoned element raises `ValidationError`, caught, logged, skipped — never crashes the cycle. |
