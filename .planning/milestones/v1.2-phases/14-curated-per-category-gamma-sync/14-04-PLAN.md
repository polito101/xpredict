| redbeat schedule in Redis ↔ beat process | The live periodic schedule is runtime state (Redis), not just code; a code swap is inert until beat reloads |

## STRIDE Threat Register

(Applicable ASVS: V5 Input Validation. No auth/session/access-control — internal Celery beat task, public read-only upstream.)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-12 | Denial of Service | malformed/oversized JSON or 5xx in ONE category crashes the cycle | mitigate | Per-category try/except keep-last-good + commit-per-category (CAT-05): a poisoned/failing category is logged + rolled back + skipped; the other six still sync; the catalog is never blanked (sync only upserts, never deletes). |
| T-14-13 | Tampering | overlapping polls double-write / interleave | mitigate | Distinct `EVENTS_LOCK_KEY` SETNX owner-token lock + Lua compare-and-delete release (WR-05); TTL 280s < 300s tick so a crashed task auto-releases before the next tick. |
