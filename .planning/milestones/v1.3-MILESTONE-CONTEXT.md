# v1.3 Live-Bets demo — Milestone Context (plan-of-record)

**Status:** ✅ Shipped 2026-06-06 (off-grid, like v1.1; merged via `171aee5`) · **Type:** Demo-only · **Started:** 2026-06-05 · **Audit:** [`v1.3-MILESTONE-AUDIT.md`](v1.3-MILESTONE-AUDIT.md)

## What
Embed **live-bets** (B2B multi-player vehicle-traffic betting, HLS-synced) inside **XPredict**, which acts as the live-bets **operator**. A player bets from a new `/live` route and their single **XPredict wallet** is debited on placement and credited on win — one balance, play-money, no real money.

**Authoritative design contract:** [`../../docs/superpowers/specs/2026-06-05-live-bets-integration-design.md`](../../docs/superpowers/specs/2026-06-05-live-bets-integration-design.md) (read this first — it has the full topology, live-bets API contract, ledger flow, and out-of-scope).

## Locked decisions
1. **Unified wallet** — XPredict ledger is the single balance; XPredict is the operator and mirrors money.
2. **Embed the widget** — `<live-bets-table>` in a new `/live` route, wrapped in XPredict chrome.
3. **Mirror by events (Approach A)** — DOM-event-driven, server-side verified via `GET /v2/bets/{id}`, idempotent by `bet_id`.
