## Acceptance

- **E2E proven:** a bet placed from `/live` moves the XPrediction wallet (debit on placement, credit on settle) through the validated ledger path — no hand-written ledger rows, 23505 session-per-call discipline preserved.
- **Real contract drift fixed** against the actual live-bets API (vs the v3 written guide): `GET /tables`, `BetView.id/selection`, `/v2/sessions` requires an `Idempotency-Key`.
- **Migration:** additive only; `livebets_bridge` (`0011_livebets_bridge`) is reversible. Post-merge it was linearized to chain after the v1.2 head `0011_phase13_market_groups` (the transient two-`0011`-heads divergence is resolved on `main` — single head verified `alembic heads` → `0011_livebets_bridge`).

## Out of scope (acknowledged, by design — demo)

Real money / PSP · OAuth client_credentials · OTEL passthrough · production webhook hardening (HTTPS/DLQ) · bulletproof cross-DB reconciliation · live-bets catalog browse / lobby.

## Notes / carry-forward

- Demo-only milestone: not production-hardened by intent. Productionizing live-bets (real webhook hardening, cross-DB reconciliation, OAuth) is a candidate future milestone.
- The `/live` surface degrades to a correct "No live table configured" empty state without the live-bets backend + widget env — safe in any environment.
