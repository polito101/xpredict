<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/CLI → POST /admin/auth/login | Untrusted form body crosses into the admin login proxy route; same untrusted-input model as the player surface |
| Admin Bearer (Authorization header) → /admin/* routes | Bearer token is the credential; FastAPI guard `current_active_admin` enforces `is_superuser=True` on every request |
| Cookie session (player) → /admin/* | Trust boundary INSIDE the app: player credentials must NEVER cross into admin authority (D-03 enforced via two FastAPIUsers instances) |
| Environment (.env.local) → bin/create_admin.py | FIRST_ADMIN_EMAIL + FIRST_ADMIN_PASSWORD cross from the operator's host into the seeding script; password is hashed before persistence |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
