Build `WalletService` (async) — the ONLY writer to the ledger — as a direct port of the
empirically-validated spike harness (`_lib/harness.py`). Pessimistic `SELECT ... FOR UPDATE`
inside `AsyncSession.begin()` (Spike 002: 1.00x amplification, the winner), one-transaction-per-transfer
with paired entries netting to zero (Spike 003 part 1), idempotency via `idempotency_key UNIQUE` →

(Spike 004: ordered → 0 deadlocks).

harness `run_load` / `LoadResult.correct` invariant (final balance exact, drift 0, CHECK rejects
overdraw) — now bound to PRODUCTION service code, not raw SQL.

Purpose: deliver the race-safe transfer engine (WAL-07) that registration (03-03), recharge (03-04),
reads (03-05), and reconciliation (03-06) all consume. This is the technical heart of the phase.
Output: `app/wallet/service.py` + the three correctness integration tests.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| concurrent async callers → one hot wallet row | The race the whole phase exists to defeat |
| service caller → transaction boundary | Caller owns commit; create_wallet must not commit early |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering | Concurrent debits double-spend / drive balance negative | mitigate | `SELECT ... FOR UPDATE` row lock inside one tx (Spike 002); test_50_concurrent_overdraft proves drift 0 |
| T-03-07 | Tampering | Replayed recharge (retry/double-click) double-credits | mitigate | catch `IntegrityError` sqlstate 23505 → return existing transfer (Spike 003); test_idempotent_recharge |
| T-03-08 | Tampering | Partial transfer leaves orphan entry / unbalanced ledger | mitigate | one `session.begin()` per transfer; fault → full rollback; test_fault_mid_transaction_rolls_back (PITFALLS #10) |
| T-03-09 | Denial of Service | Deadlock (40P01) on multi-account locks | mitigate | canonical UUID lock order `sorted(..., key=str)` (Spike 004 → 0 deadlocks) |
| T-03-10 | Tampering | create_wallet commits early, breaking the SC#1 single-tx contract | mitigate | create_wallet add+flush only (mirror AuditService.record); enforced by grep + 03-03 test |
</threat_model>
