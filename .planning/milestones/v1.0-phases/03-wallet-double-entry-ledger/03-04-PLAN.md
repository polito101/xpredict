<objective>
Ship the admin recharge primitive (SC#3, WAL-09 firewall): `POST /admin/wallets/{user_id}/recharge`
behind the Phase 2 admin Bearer gate (`current_active_admin`), taking an `Idempotency-Key` header.
It debits `house_promo` then credits the path user's `user_wallet` via `WalletService.recharge`
(built in 03-02), and is idempotent: a second identical call returns the same transfer id with no
double-credit (Spike 003 part 2; 23505 then return existing).

Also lock the SC#5 regulatory firewall here, where the only wallet-mutation API lives: the
`RechargeRequest` schema uses `extra="forbid"` so a `dst_user_id`-style param is a hard 422, and a
negative test asserts no user-to-user transfer path exists (no API param, no schema FK).

Purpose: the first money-moving endpoint, idempotent and admin-only, with the no-transfer guard
(PITFALLS #3) observable at the schema boundary.
Output: `app/wallet/{schemas,admin_router}.py`, the router wiring in `app/main.py`, and the
recharge plus no-user-to-user tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin client -> recharge endpoint | Untrusted Bearer + body cross here; must be admin + schema-validated |
| recharge body -> ledger | A rogue destination field would be a user-to-user move (regulatory breach) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-13 | Elevation of Privilege | Player recharges own/other wallet | mitigate | endpoint gated by `current_active_admin` (Bearer + is_superuser); test_recharge_requires_admin |
| T-03-14 | Tampering | Replayed recharge double-credits | mitigate | `Idempotency-Key` + service 23505 dedup; test_recharge_idempotent_same_key (SC#3) |
| T-03-15 | Elevation / regulatory | user-to-user value transfer via a dst_user_id param | mitigate | `RechargeRequest` `extra="forbid"`; no destination field; no Entry FK to a second user wallet (WAL-09 / PITFALLS #3); test_no_user_to_user |
| T-03-16 | Information Disclosure | Money leaked as float | mitigate | `MoneyStr` serializer; test_recharge_amount_is_string_in_json (SC#4 at recharge surface) |
| T-03-17 | Repudiation | Admin recharge with no audit trail | mitigate | `AuditService.record(event_type="wallet.recharge")` in the same tx as the transfer |
</threat_model>
