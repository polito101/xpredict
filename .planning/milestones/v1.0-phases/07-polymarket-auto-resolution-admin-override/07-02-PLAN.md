# Plan 07-02: Auto-Resolution Beat Task (STL-01)

## Objective
Implement the `detect_polymarket_resolutions` Celery Beat task (60s) that auto-settles mirrored Polymarket markets via the unchanged `SettlementService.resolve_market`, gated by the canonical `_derive_status()` state machine plus the `uma_resolved_at` grace period.

## Context
This is the core of STL-01. Per `07-RESEARCH.md`, Phase 7's value is in the WIRING, not new logic — every piece of infrastructure already exists. The task assembles them under the critical invariant: NEVER settle on `closed=true` alone; only settle when `_derive_status()` returns `RESOLVED` AND the configurable grace period has elapsed. It depends on Plan 07-01 (the `uma_resolved_at` column and `POLYMARKET_GRACE_PERIOD_MINUTES` setting must already exist and be migrated). It runs in Wave 2 in parallel with Plan 07-03 (no shared files). The Beat task follows the exact `asyncio.run(_run_*())` wrapper + injectable-override pattern from Phase 6's `poll_polymarket_top25`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Gamma API → Beat worker | Untrusted external resolution data crosses into our settlement trigger |
| Beat worker → SettlementService (ledger) | Money-moving call; must be idempotent and correctly gated |

## Security Threats

| Threat ID | Threat | STRIDE | Disposition | Mitigation |
|-----------|--------|--------|-------------|------------|
| T-07-03 | Premature settlement on `closed=true` before UMA confirms (payouts before dispute window closes) | Tampering | mitigate | Parse every Gamma response through `GammaMarket.model_validate(raw)` and only proceed when `parsed.internal_status == MarketStatus.RESOLVED`. Never read `raw["closed"]` directly. Guarded by `test_closed_proposed_not_settled` (SC#3). |
| T-07-04 | Double-settlement under Celery at-least-once delivery (two overlapping ticks settle the same market) | Tampering | mitigate | Conditional UPDATE `WHERE uma_resolved_at IS NULL` so only one writer starts the clock (constraint 3); `SettlementService` idempotency guard (`WHERE bets.status = PENDING`) is the second net. Optional distinct Redis lock `xpredict:detect:polymarket:lock` (never reuse the poll lock). |
| T-07-05 | Settling the wrong outcome from ambiguous `outcomePrices` | Tampering | mitigate | Map winner via label cross-reference `_map_winning_outcome_id(outcome_prices_raw, outcomes_raw, db_outcomes)` (constraint 8), raising `ValueError` when no clear winner — never silently guess. |
| T-07-06 | Grace-period state lost on Redis flush, silently restarting the clock | Denial of Service | mitigate | Grace state is the `uma_resolved_at` DB column (constraint 2), not Redis — survives restarts. |
</threat_model>
