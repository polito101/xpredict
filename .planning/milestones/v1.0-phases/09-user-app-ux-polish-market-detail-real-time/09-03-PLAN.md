## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser WS client → backend `/ws/markets/{id}` | The client opens an unauthenticated read-only price socket |
| npm registry → build | Four new packages enter the dependency tree + lockfile |
| browser env → client bundle | `NEXT_PUBLIC_WS_URL` is exposed to the browser bundle by design |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-SC | Tampering | npm installs (recharts, react-is, @radix-ui/react-dialog, @radix-ui/react-select) | mitigate | Blocking human re-verification checkpoint BEFORE install (Task 1): all four are first-party/established with official repos (RESEARCH Package Legitimacy Audit — Approved, none `[ASSUMED]`/`[SUS]`/`[SLOP]`). The pnpm override pins `react-is` to a single version, removing the transitive-version ambiguity that also has a supply-chain dimension. Lockfile committed. |
| T-09-09 | Information Disclosure | client WS hook | accept | The hook only RECEIVES public odds deltas (`{outcome_id, odds, ts}`) — no credentials are sent (CORS/Authorization headers do not apply to the browser WebSocket handshake; the socket is public by design — RESEARCH Pattern 4). No PII flows to the client. |
| T-09-10 | Information Disclosure | `NEXT_PUBLIC_WS_URL` in the bundle | accept | The WS base URL is non-secret (it is the public API host with a `ws://`/`wss://` scheme). Using the `NEXT_PUBLIC_` prefix is intentional — the server-only `BACKEND_URL` (secrets) is NOT used in the client hook (SP-7 / Pitfall 7). |
| T-09-11 | Denial of Service | client reconnect loop | mitigate | Reconnect uses exponential backoff capped at 30s + jitter (RESEARCH Pattern 6 / spike) — no thundering-herd tight-loop reconnect against the backend WS endpoint. |
</threat_model>
