<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer machine → git remote | every push runs through pre-commit (local) + GitHub Actions (remote); both must agree |
| public PR → main | every PR runs the 3 CI workflows; main is protected (per CLAUDE.md branch policy) — PRs only merged by Pol |
| GitHub Actions runner → Sentry SaaS | CI runs do NOT init Sentry (no `SENTRY_DSN` in CI env); Sentry events are dev/staging/prod surface only |
| .gitleaks.toml allowlist → security risk | allowlist permits `tests/.*fixtures.*` and `docs/.*\.md` — the synthetic-secret fixture lives in this allowlist exactly so the developer CAN commit it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | Real secret committed to git (T-01 from phase model) | mitigate | gitleaks runs in (a) pre-commit `gitleaks protect --staged` — fast diff scan; (b) GitHub Actions on every PR — full HEAD scan; (c) weekly full-history scan via `schedule: cron`; custom rules for `SESSION_SIGNING_KEY` and `ADMIN_TOKEN` extend the default ruleset (D-33); `.env.local` gitignored (D-32) |
| T-04-02 | Information Disclosure | gitleaks allowlist for test fixtures becomes too broad → real secret slips through (Pitfall 6) | mitigate | Allowlist regex is specific: `tests/.*fixtures.*` (NOT `tests/.*`); fixture files have an explicit `.gitleaksignore-note.md` documenting "this directory contains known-fake secrets for testing the linter — never put real secrets here" |
| T-04-03 | Tampering | Pre-commit hook bypassed via `--no-verify` accidentally | accept | Local hook is recommendation; CI workflows are the actual gate (cannot bypass); PR-protection rules prevent merging without green CI |
| T-04-04 | Repudiation | Money-column lint passes locally but fails in CI due to env differences | mitigate | CI invokes the lint via the same `uv run python scripts/lint_money_columns.py` command as pre-commit; both run on Python 3.12 with the same `pyproject.toml` deps; CI uses `uv sync --frozen` so deps match exactly |
| T-04-05 | Information Disclosure | docker-compose `xpredict:xpredict` credentials accidentally end up in CI logs | accept | The credentials are placeholders documented in `.env.example` (T-03-02); CI does NOT run docker-compose with real DB credentials; integration tests use testcontainers which auto-generate ephemeral credentials |
| T-04-SC | Tampering | Phase 1 npm/pip installs without legitimacy audit (supply-chain attack) | mitigate | 01-RESEARCH §"Package Legitimacy Audit" ran slopcheck on every package; all 38 packages returned `OK`; this plan inherits the audit — no new packages installed beyond what 01-01/01-02 already installed |
