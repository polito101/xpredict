<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → process | `.env.local` (gitignored) and `.env.example` (committed placeholders) read by Pydantic Settings at boot |
| docker-compose internal network | backend ↔ db ↔ redis on the compose bridge (plaintext); no untrusted client traffic in Phase 1 |
| public HTTP | `/healthz`, `/readyz`, `/_sentry-test` exposed unauthenticated — Phase 1 ships no auth surface |
| Sentry SDK → Sentry SaaS | events leave the host with `service=` tag and `release=` env; PII suppressed via `send_default_pii=False` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Settings loading `.env` files | mitigate | `extra="ignore"` on `SettingsConfigDict` so future env vars don't ValidationError-leak names (Pitfall 3); reads via typed `BaseSettings` only — no `os.getenv()` elsewhere; structlog `scrub_secrets` processor drops `password`/`password_hash`/`session_signing_key`/`admin_token`/`sentry_dsn`/`api_key`/`secret`/`xp_session` keys (D-25) |
| T-01-02 | Information Disclosure | Sentry events leaving the process | mitigate | `send_default_pii=False` on every `sentry_sdk.init()` call (Pattern 5a); `release` env defaulted to `dev-{git-sha}` locally; `service` tag set immediately after init so events filter correctly |
| T-01-03 | Tampering | Money columns in Phase 3+ might use Float (no `audit_log` yet, but the discipline is locked here) | mitigate | `Money` Annotated alias forces `Numeric(18, 4)`; AST lint `scripts/lint_money_columns.py` walks every `*models*.py` and fails on Float/wrong-precision Numeric for money-named columns (D-17); 4 unit tests cover pass/fail/wrong-args/warn cases; lint runs in pre-commit + CI |
| T-01-04 | Repudiation | Sentry events arrive without `service=` tag, can't be filtered | mitigate | Pitfall 5: init MUST happen inside `worker_process_init`/`beat_init` signals (not module-level); tag set immediately after init; tests assert init function is called and `set_tag("service", ...)` is invoked |
| T-01-05 | Denial of Service | `/healthz` or `/readyz` leaking DB connection strings on failure | mitigate | `/healthz` returns `{"status":"ok"}` only — no dependency info; `/readyz` returns `{"failures": {"db": <truncated error>}}` only on 503; never echoes connection strings or version info (D-30) |
| T-01-06 | Tampering | structlog contextvars leak across Celery tasks (Pitfall 7) | mitigate | Wire `task_prerun` to `structlog.contextvars.clear_contextvars()` + bind `task_id`/`task_name`; `task_postrun` to clear again — explicitly tested in 01-03 |

</threat_model>
