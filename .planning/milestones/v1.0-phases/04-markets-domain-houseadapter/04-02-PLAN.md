<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> admin API | Admin Bearer JWT required; untrusted input must be validated by Pydantic schemas |
| client -> public API | No auth; read-only endpoints; no write capability exposed |
| admin input -> DB | Pydantic validates all fields; SQLAlchemy parameterized queries prevent injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Elevation of Privilege | admin_market_router | mitigate | All admin routes use Depends(current_active_admin); negative test confirms 401 (no token) and 403 (non-admin token) |
| T-04-05 | Tampering | MarketUpdate | mitigate | Explicit MarketUpdate Pydantic schema lists only editable fields (resolution_criteria, deadline, odds_yes, category); criteria-lock check before any update; no mass assignment possible |
| T-04-06 | Tampering | SQL injection via filter params | mitigate | Filter params (source, status, category) validated as Pydantic enum or str with max_length; all queries use SQLAlchemy parameterized queries, never string interpolation |
| T-04-07 | Information Disclosure | public market list | accept | Public endpoint intentionally returns OPEN market data (question, odds, deadline); this is the business requirement. No sensitive data (admin IDs, internal IDs) exposed in public responses. |
| T-04-08 | Spoofing | slug collision | mitigate | Slug generated with UUID suffix + UNIQUE constraint on slug column; collision probability negligible; 404 on lookup failure, never wrong market |
| T-04-09 | Repudiation | market mutations | mitigate | AuditService.record called on every create/update/close with actor UUID, event_type, payload, and IP; immutable audit_log table (Phase 1 trigger) |
| T-04-SC | Tampering | npm/pip installs | mitigate | No new packages in Plan 02; python-slugify installed in Plan 01 |
</threat_model>
