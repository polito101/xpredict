<objective>
Wire per-market stake limits (BET-06) end-to-end: carry the new `markets.min_stake`/`max_stake` columns (added by 12-01) through the bets read port into `place_bet`'s server check, and extend the client order-form mirror to prefer the per-market bounds. NULL columns fall back to the existing global config — no behavior change for existing markets.

Purpose: The v1.0 audit found BET-06 only partially built — global min/max enforced dual-side, but the "configurable PER-MARKET" clause was never wired (there's a literal code comment in `bets/router.py` deferring it to "Phase 10 (TenantConfig)" that was never executed). This plan executes it on the columns 12-01 added (per the columns-vs-TenantConfig divergence documented in 12-01).

Output: `MarketView` with nullable `min_stake`/`max_stake`, the production read adapter populating them, the per-market check moved into `BetService.place_bet` (where the validated market is available), the global fallback, extended backend + client tests, and the order-form client mirror reading per-market bounds.

Decision (RESEARCH A4): the per-market check moves INTO `BetService.place_bet` (right after the market is validated, `service.py` ~L84-88) because the router does NOT have the market loaded — only the service fetches it via the port. The current router-level global check (`bets/router.py:90-97`) is superseded by the service check; remove/replace it so the limit is checked once, with the market in hand.
</objective>
