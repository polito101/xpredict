<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| player browser to public /branding/* | Untrusted, unauthenticated read of operator-chosen branding. |
| admin browser (Bearer) to /api/v1/admin/tenant-config | Authenticated admin write; operator-controlled palette + logo bytes cross here. |
| stored hex/logo to player <style>/<img> | Operator-controlled values are later rendered into the player DOM (Plan 10-05) — validation must happen at THIS persist boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering / Elevation | primary_hex / secondary_hex persisted then injected into a player `<style>` block (Plan 10-05) | mitigate | Server-side strict allowlist `^#[0-9a-fA-F]{6}$` via Pydantic `Field(pattern=...)` on TenantConfigUpdate BEFORE persist; a 6-digit hex cannot contain `<`, `>`, `}`, or quotes, so no `</style>` break-out is possible. Reject non-conforming with 422. |
| T-10-02 | Elevation (XSS) | logo SVG payload served same-origin | mitigate | Serve logo via GET /branding/logo with the stored Content-Type + `X-Content-Type-Options: nosniff`; the player consumes it via `<img src>` (SVG-in-img does not execute script). SVG accepted only under the size cap and never inlined into the DOM. |
| T-10-03 | Denial of Service | oversized / image-bomb logo upload | mitigate | Hard size cap (<=256KB / 262144 bytes) enforced server-side; reject with 422. No image resizing/decoding (no Pillow) — avoids decompression bombs. |
| T-10-04 | Spoofing / Tampering | content-type lies (e.g. an HTML payload declared image/png) | mitigate | Content-Type allowlist + leading magic-byte verification for PNG/JPEG/WebP; declared-type mismatch rejected 422. |
| T-10-05 | Elevation | a player reaches /api/v1/admin/tenant-config | mitigate | `Depends(current_active_admin)` on both GET and PUT; SC#6 negative test (player Bearer → 403, no Bearer → 401) in test_tenant_config_negative.py. |
| T-10-06 | Information Disclosure | public /branding/current leaks sensitive fields | mitigate | BrandingPublic returns exactly brand_name + 2 hexes + logo_url; no bytes, no tenant_id, no timestamps. test_branding_public.py asserts the exact field set. |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No new packages this phase (RESEARCH §Package Legitimacy Audit — not applicable; zero new deps). No install task exists; no legitimacy checkpoint required. |
</threat_model>
